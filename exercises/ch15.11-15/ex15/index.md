### クロスオリジンリクエストに制約が無かった場合、どのような問題が発生するか
- 参考：https://zenn.dev/mukkun69n/articles/8a04939fd66a4a
- 参考：https://qiita.com/att55/items/2154a8aad8bf1409db2b

- 仮にどのオリジンからも通信を許可するとする
    - →悪意のあるウェブサイトから、ユーザーのデータや情報を不正にアクセスすることが可能になってしまう
- 具体的には以下の脆弱性がある
    - XSS：ユーザーが Web サイトにアクセスすることで不正なスクリプトが Client (Web ブラウザ) で実行されてしまう脆弱性。Cookie 内のセッション情報を抜き取られて不正ログインを行われるなど
    - CSRF (Cross-Site Request Forgeries)
        - Web アプリケーションのユーザーが、意図しない処理を Web アプリケーション (Web Server) 上で実行される脆弱性
### クロスオリジンリクエストで メソッド(`POST`/`GET`)やリクエストの内容によって Preflight リクエストの有無が異なるのは何故か
- 参考：https://zenn.dev/tm35/articles/ad05d8605588bd
- Post,PUT,DELETEなどのように、データが変更されるような副作用が発生する可能性がある
- 悪意あるサイトからこれらのリクエストが送られてきた場合、ブラウザがレスポンスを利用できなくても副作用が発生させられる可能性がある
- よって、Preflight を送信して Response を確認する必要がある
