- 複数のTCPクライアント (net.Socket) でHTTPリクエストを送信せず同時に接続を維持した際、何接続で接続が確立できなくなるか
    - 確認方法：client.jsを作成し、connect数を増やしていった
    - 結果：160000接続あたりでコンソールが止まる(＝詰まった？)
    - 理由：
        - 参考：https://qiita.com/khayama/items/1830580c669c0511e73a
        - Windows Serverにおけるポート枯渇問題による
        - `netsh int ipv4 show dynamicport tcp`で利用できるポート範囲(エフェメラルポート)を確認すると、以下が表示された
            ```
            プロトコル tcp の動的ポートの範囲
            ---------------------------------
            開始ポート      : 49152
            ポート数        : 16384
            ```
        - エフェメラルポートとはOSがクライアント通信のたび自動割り当てするポート番号
        - Windowsで使えるポート範囲は上記より16384個
        - よって、接続数が16000個あたりになると接続が不安定or不可能になる
