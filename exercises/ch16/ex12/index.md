### C10K問題とは
- 参考：https://ja.wikipedia.org/wiki/C10K%E5%95%8F%E9%A1%8C
- 参考：https://zenn.dev/kimifan/articles/c-10k-problem
- クライアント1万問題の略
- Apache HTTP ServerなどWebサーバーとクライアントの通信において、クライアントが訳1万台に達すると、ハードウェア的には余裕がある(CPUは暇)なのにレスポンス性能が著しく下がる問題(OSが処理しきれなくなる)
- 原因としてApache HTTP Serverの駆動方式にある
    - 1接続1プロセス/1スレッドの仕様となっている→各スレッドはメモリを食う
    - 大量のクライアントからアクセスがあるとその数だけプロセスを生成する必要がある
    - しかしWindowsやUnix系OSでは、同時に起動できるプロセスには32767(215-1)個の制限がある。
    - よって、この制限を超えると新たな接続は待たなければなくなる
- 回避法
    - Nginxを使う
    - サーバサイドでNode.jsなど駆動方式を持ったソフトウェアを使用する
        - レスポンスなどを待つ時間、スレッドの占有をやめる
        - シングルスレッド、イベント駆動、ノンブロッキングI/Oモデルという特徴から他の処理お同時進行で行うことができる
